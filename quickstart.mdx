Get Neutrino running locally with both the backend API and frontend dashboard. This guide walks through environment setup, dependency installation, and running all services.

## Prerequisites

Before starting, ensure you have:

- **Python 3.11+** with [uv](https://github.com/astral-sh/uv) package manager
- **Node.js 18+** and npm
- **Supabase account** (for database and pgvector)
- **OpenRouter API key** (for LLM access)
- **Serper API key** (optional, for trend data collection)

## Environment setup

Neutrino requires three environment variables to function. Create a `.env` file in the project root:

```env
# Required: Database connection
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

# Required: LLM access (supports up to 4 keys for rotation)
OPENROUTER_API_KEY_1=sk-or-v1-xxx
OPENROUTER_API_KEY_2=sk-or-v1-xxx  # Optional
OPENROUTER_API_KEY_3=sk-or-v1-xxx  # Optional
OPENROUTER_API_KEY_4=sk-or-v1-xxx  # Optional

# Optional: Trend intelligence
SERPER_API_KEY=your-serper-api-key
```

<Note>
API keys are rotated automatically when multiple keys are provided. This prevents rate limiting during high-volume analysis.
</Note>

## Backend setup

<Steps>
  <Step title="Navigate to backend directory">
    ```bash
    cd backend
    ```
  </Step>

  <Step title="Install dependencies with uv">
    ```bash
    uv sync
    ```

    This installs all Python dependencies defined in `pyproject.toml` and creates a virtual environment.
  </Step>

  <Step title="Set up database schema">
    Create the `analysis_records` table in Supabase:

    1. Open the Supabase SQL Editor
    2. Copy the migration from `backend/migrations/001_create_analysis_records.sql`
    3. Execute the SQL to create the table with pgvector extension

    The schema includes fields for tech stack, risks, recommendations, and trend data.
  </Step>

  <Step title="Start the FastAPI server">
    ```bash
    uv run uvicorn advisor.api.endpoints:app --reload --port 8000
    ```

    The API server will start at `http://localhost:8000` with auto-reload enabled for development.
  </Step>

  <Step title="(Optional) Start Streamlit UI">
    In a separate terminal:

    ```bash
    cd backend
    uv run streamlit run src/advisor/ui/app.py
    ```

    The Streamlit interface launches at `http://localhost:8501` and provides a simplified UI for repository analysis.
  </Step>
</Steps>

<Info>
The FastAPI server provides interactive API documentation at `http://localhost:8000/docs` powered by Swagger UI.
</Info>

## Frontend setup

<Steps>
  <Step title="Navigate to frontend directory">
    ```bash
    cd frontend
    ```
  </Step>

  <Step title="Install npm dependencies">
    ```bash
    npm install
    ```

    This installs React 19, Vite, Tailwind CSS, Framer Motion, and other UI dependencies.
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    The React application will launch at `http://localhost:5173` with hot module replacement.
  </Step>
</Steps>

## Test the API

Once the backend is running, test the `/analyze` endpoint with a real repository:

<CodeGroup>
```bash curl
curl -X POST http://localhost:8000/analyze \
  -H "Content-Type: application/json" \
  -d '{
    "repo_url": "https://github.com/astralks/lstm_from_scratch"
  }'
```

```bash curl (Private Repo)
curl -X POST http://localhost:8000/analyze \
  -H "Content-Type: application/json" \
  -d '{
    "repo_url": "https://github.com/your-org/private-repo",
    "access_token": "ghp_your_github_token"
  }'
```

```python Python
import requests

response = requests.post(
    "http://localhost:8000/analyze",
    json={
        "repo_url": "https://github.com/astralks/lstm_from_scratch"
    }
)

result = response.json()
print(f"Analysis ID: {result['analysis_id']}")
print(f"Technical Summary: {result['technical_summary'][:200]}...")
```

```javascript JavaScript
fetch('http://localhost:8000/analyze', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    repo_url: 'https://github.com/astralks/lstm_from_scratch'
  })
})
  .then(res => res.json())
  .then(data => {
    console.log('Analysis ID:', data.analysis_id);
    console.log('Technical Summary:', data.technical_summary);
  });
```
</CodeGroup>

<Warning>
GitHub access tokens for private repositories are **ephemeral** — they are used only for the request and never stored in the database.
</Warning>

## Performance expectations

Typical analysis times vary by repository size:

- **Small repos** (< 50 files): 30-45 seconds
- **Medium repos** (50-200 files): 45-60 seconds  
- **Large repos** (200+ files): 60-90 seconds

The timeline includes:
- **GitHub fetch**: 2-5 seconds (strategic 3-pass file fetching)
- **Tech stack extraction**: < 1 second (in-memory static analysis)
- **Trend search**: 5-15 seconds (RAG cache hits are instant; misses trigger parallel web searches)
- **Deep review**: 25-40 seconds (3 parallel AI calls with 128K context windows)

<Tip>
For faster results, ensure trend data is pre-cached. Run a batch collection script to populate the RAG store with common technologies before analyzing repositories.
</Tip>

## Troubleshooting

<Accordion title="Port already in use">
  If port 8000 or 5173 is occupied, specify a different port:

  ```bash
  # Backend on port 8080
  uv run uvicorn advisor.api.endpoints:app --reload --port 8080

  # Frontend on port 3000
  npm run dev -- --port 3000
  ```
</Accordion>

<Accordion title="Missing environment variables">
  If you see `ValidationError` on startup, ensure your `.env` file contains all required variables:

  ```bash
  # Check which variables are loaded
  uv run python -c "from advisor.config import get_settings; print(get_settings())"
  ```

  The settings validator will show which variables are missing.
</Accordion>

<Accordion title="Database connection fails">
  Verify your Supabase credentials:

  1. Check that `SUPABASE_URL` matches your project URL
  2. Use the **service role key**, not the anon key
  3. Ensure the `analysis_records` table exists (run the migration)
  4. Check that the pgvector extension is enabled in Supabase
</Accordion>

<Accordion title="Analysis times out or takes too long">
  Several optimizations can reduce analysis time:

  - **Reduce max files**: Edit `orchestrator.py` to fetch 150 files instead of 300
  - **Cache trends**: Pre-populate the RAG store to avoid fresh searches
  - **Use faster models**: Switch to a lower-latency model in `llm/models.py`
  - **Increase API keys**: Add more OpenRouter keys for rotation (up to 4)
</Accordion>
