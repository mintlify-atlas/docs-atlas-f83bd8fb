The analysis engine is Neutrino's core intelligence system. It coordinates repository intake, parallel AI code review, and evidence-based report generation.

## Architecture overview

The analysis engine uses a three-phase pipeline:

1. **GitHub intake** - Fetch up to 300 files with strategic prioritization
2. **Deep AI review** - Parallel analysis with 3 specialized agents (backend, frontend, infrastructure)
3. **Report aggregation** - Synthesis into technical and executive summaries

<Note>
The orchestrator in `backend/src/advisor/analysis/core/orchestrator.py:64-153` coordinates all analysis phases with timeline tracking.
</Note>

### Version extraction

The detector extracts version information from package files:

```python
# From package.json
if "version" in pkg:
    versions["project"] = pkg["version"]

# Framework versions
deps = {**pkg.get("dependencies", {}), **pkg.get("devDependencies", {})}
for key in ["react", "vue", "angular", "next", "typescript"]:
    if key in deps:
        versions[key] = deps[key].lstrip("^~")
```

### Confidence scoring

The analyzer calculates confidence based on pattern matches:

```python
for pattern in arch_info["patterns"]:
    if pattern in directories:
        evidence.append(f"Found '{pattern}' directory")
        matches += 1

# Confidence = matches / total_patterns
confidence = min(matches / len(arch_info["patterns"]), 1.0)

# Only report patterns with >= 30% confidence
if confidence >= 0.3:
    patterns.append(ArchitecturePattern(
        pattern_name=arch_info["name"],
        confidence=round(confidence, 2),
        evidence=evidence,
    ))
```

## Parallel deep review agents

The `DeepReviewOrchestrator` runs 3 parallel AI agents for comprehensive code analysis.

### Smart chunking strategy

From backend/src/advisor/analysis/core/deep_review.py:1-12:

```python
# Architecture:
# 1. Fetch entire repo → optimize tokens
# 2. Partition into frontend, backend, infra buckets
# 3. Backend: send full logic + imports (priority)
# 4. Frontend/Infra: send function names + imports only (lightweight)
# 5. If frontend+infra < 60K tokens → single combined AI call
# 6. Aggregate findings into evidence-based reports
```

### Token optimization

The engine compresses files before AI analysis:

```python
optimizer = TokenOptimizer(
    max_file_chars=15000,    # Truncate long files
    max_total_chars=700000,  # Stay under 128K token limit
)

optimized, stats = optimizer.optimize(file_contents)
logger.info(
    f"Optimized: {stats.original_chars:,} -> {stats.compressed_chars:,} "
    f"({stats.savings_percent:.1f}% saved)"
)
```

### Parallel execution

The orchestrator runs agents concurrently:

```python
# Option 1: Combined call (frontend + infra together)
if combined_chars < COMBINED_THRESHOLD_CHARS:
    results = await asyncio.gather(
        analyze_chunk("backend", backend_prompt, backend_files),
        analyze_chunk("frontend+infra", combined_prompt, all_files),
        return_exceptions=True,
    )

# Option 2: Separate calls (3 parallel agents)
else:
    results = await asyncio.gather(
        analyze_chunk("backend", backend_prompt, backend_files),
        analyze_chunk("frontend", frontend_prompt, frontend_files),
        analyze_chunk("infrastructure", infra_prompt, infra_files),
        return_exceptions=True,
    )
```

<Warning>
The engine maximizes use of 128K context windows. Backend code gets full content, while frontend/infrastructure get function signatures only to stay within token limits.
</Warning>

Every analysis phase is tracked for observability:

```python

try:
    structure, files = await fetch_repository(owner, repo)
    timeline.complete_phase("github_fetch")
except Exception as e:
    timeline.fail_phase("github_fetch", str(e))
    raise

# Access timeline data
```
