## GET /analyses

Retrieve a list of recent repository analyses.

This endpoint returns the most recent analyses stored in the database, ordered by creation time (newest first).

## Request

<ParamField query="limit" type="number" default="20">
  Maximum number of analyses to return.
  
  Must be a positive integer. Defaults to 20 if not specified.
</ParamField>

### Request example

```bash
curl https://api.neutrino.example.com/analyses
```

With limit parameter:

```bash
curl https://api.neutrino.example.com/analyses?limit=50
```

## Response

Status code: `200 OK`

<ResponseField name="analyses" type="array" required>
  Array of `AnalysisRecord` objects. Each record contains the complete analysis data as described in the [Get analysis](/api/get-analysis) endpoint.
  
  Ordered by creation time, newest first.
</ResponseField>

<ResponseField name="count" type="number" required>
  Total number of analyses returned in this response.
</ResponseField>

### Empty response

When no analyses are found:

```json
{
  "analyses": [],
  "count": 0
}
```

## Error responses

### 500 Internal Server Error

Returned when there is a database error:

```json
{
  "detail": "Failed to list analyses: Database connection error"
}
```

## Notes

- If the database table does not exist, the endpoint returns an empty list instead of an error
- Results are always ordered by creation time (newest first)
- Each analysis record in the array contains the full data structure as returned by the [Get analysis](/api/get-analysis) endpoint
