---
title: Analysis record
description: Complete analysis record stored in the database
---

The `AnalysisRecord` model represents the complete analysis result for a repository. This is the single source of truth for all analysis data, from which PDF reports are derived on demand.

## Fields

<ResponseField name="id" type="UUID | None">
  Unique identifier for the analysis record. Auto-generated by the database if not provided.
</ResponseField>

<ResponseField name="repo_url" type="string" required>
  URL of the analyzed GitHub repository.
</ResponseField>

<ResponseField name="repo_name" type="string" required>
  Name of the repository (e.g., "owner/repo").
</ResponseField>

<ResponseField name="analyzed_at" type="datetime" required>
  Timestamp when the analysis was performed. Defaults to current UTC time.
</ResponseField>

<ResponseField name="model_used" type="string" required>
  Name or identifier of the AI model used for analysis.
</ResponseField>

### Structured analysis output

<ResponseField name="tech_stack" type="TechStackInfo" required>
  Detected technology stack information.
  
  <Expandable title="TechStackInfo fields">
    <ResponseField name="languages" type="list[string]">
      Programming languages detected in the codebase.
    </ResponseField>
    
    <ResponseField name="frameworks" type="list[string]">
      Frameworks and libraries used.
    </ResponseField>
    
    <ResponseField name="databases" type="list[string]">
      Database systems identified.
    </ResponseField>
    
    <ResponseField name="tools" type="list[string]">
      Development and build tools.
    </ResponseField>
    
    <ResponseField name="package_managers" type="list[string]">
      Package managers in use.
    </ResponseField>
    
    <ResponseField name="versions" type="dict[string, string]">
      Version information for detected technologies (key: technology name, value: version string).
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="architecture_patterns" type="list[ArchitecturePattern]">
  Architecture patterns detected in the codebase.
  
  <Expandable title="ArchitecturePattern fields">
    <ResponseField name="pattern_name" type="string" required>
      Name of the detected architecture pattern.
    </ResponseField>
    
    <ResponseField name="confidence" type="float" required>
      Confidence score between 0.0 and 1.0.
    </ResponseField>
    
    <ResponseField name="evidence" type="list[string]">
      List of evidence supporting the pattern detection.
    </ResponseField>
    
    <ResponseField name="description" type="string">
      Description of how the pattern is implemented.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="risks_and_gaps" type="list[RiskItem]">
  Identified risks and gaps in the codebase. See [RiskItem](/api/models/risk-item) for details.
</ResponseField>

<ResponseField name="recommendations" type="list[Recommendation]">
  Forward-looking recommendations for improvement. See [Recommendation](/api/models/recommendation) for details.
</ResponseField>

### Enhanced analysis output

<ResponseField name="features" type="list[Feature]">
  User-facing features detected in the codebase.
  
  <Expandable title="Feature fields">
    <ResponseField name="name" type="string" required>

    <ResponseField name="description" type="string">

  </Expandable>

    <ResponseField name="auth_providers" type="list[string]">

    <ResponseField name="payment_integrations" type="list[string]">
      Payment processors integrated.
    </ResponseField>
    
    <ResponseField name="monetization_type" type="string">
      Type of monetization model (e.g., "freemium", "subscription", "usage-based", "one-time").
    </ResponseField>
    
    <ResponseField name="monetization_signals" type="list[string]">
      Signals indicating monetization approach.

    <ResponseField name="growth_mechanisms" type="list[string]">
      Growth and acquisition mechanisms identified.
    </ResponseField>
    
    <ResponseField name="revenue_drivers" type="list[string]">

    </ResponseField>
    
    <ResponseField name="user_tiers" type="list[string]">
      User tiers or plan levels (e.g., "free", "pro", "enterprise").
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="integrations" type="list[Integration]">
  External integrations detected in the codebase.
  
  <Expandable title="Integration fields">
    <ResponseField name="name" type="string" required>
      Name of the integration.
    </ResponseField>
    
    <ResponseField name="category" type="string" required>
      Category of integration (e.g., "cloud", "payment", "auth", "monitoring", "analytics", "communication").
    </ResponseField>
    
    <ResponseField name="detected_from" type="string">
      Source or method of detection.
    </ResponseField>
    
    <ResponseField name="description" type="string">
      Description of the integration's purpose.
    </ResponseField>
    
    <ResponseField name="cost_tier" type="string">
      Estimated cost tier (e.g., "free", "low", "medium", "high", "enterprise").
    </ResponseField>
  </Expandable>
</ResponseField>

### Summary outputs

<ResponseField name="technical_summary" type="string" required>
  Technical summary of the analysis, suitable for engineering audiences.
</ResponseField>

<ResponseField name="executive_summary" type="string" required>
  Executive summary of the analysis, suitable for business stakeholders.
</ResponseField>

### Metadata

<ResponseField name="analysis_duration_ms" type="integer | None">
  Total duration of the analysis in milliseconds.
</ResponseField>

<ResponseField name="file_count" type="integer | None">
  Total number of files in the repository.
</ResponseField>

<ResponseField name="files_analyzed" type="integer | None">
  Number of files actually analyzed (with content fetched).
</ResponseField>

<ResponseField name="token_usage" type="dict[string, integer]">
  Token usage statistics for the analysis (key: metric name, value: token count).
</ResponseField>

<ResponseField name="timeline" type="dict[string, Any] | None">
  Phase-level timestamps tracking analysis progress.
</ResponseField>

<ResponseField name="trend_data" type="dict[string, Any] | None">
  Trend intelligence context and insights.
</ResponseField>

<ResponseField name="created_at" type="datetime">
  Timestamp when the record was created. Defaults to current UTC time.
</ResponseField>

<ResponseField name="updated_at" type="datetime">
  Timestamp when the record was last updated. Defaults to current UTC time.
</ResponseField>
